# ‚≠ê 1. What is ULM?

ULM (User Lifecycle Management) is a **central identity and subscription management platform** used by Vodafone (VF) Italy and Germany to manage:

* User registration
* Onboarding (user creation + verification)
* Contract / no-contract customers
* Accounts, subscriptions, and groups
* Authentication (Web, Admin, Retail users)
* User sessions and security
* Integration with VF internal systems
* Data synchronization with CRM/ACRM
* Kafka-based real-time sync
* MIGRATION (Legacy ‚Üí Next Platform)

ULM acts as the **single source of truth** for Web user data.

---

# ‚≠ê 2. Types of Users Managed

### **1. Web User (Real VF Customer)**

* Purchases SIM, number, subscription
* Stored in ULM database
* Login:

  * 1FA ‚Üí Username/Password
  * 2FA ‚Üí Password + OTP
  * MFA ‚Üí OTP + eventClass code
* Session timeout ‚Üí 30 minutes (inactive)

### **2. Retail User (Shop / SIM Retailer)**

* Has limited admin permissions
* Not stored in ULM DB
* Login via **SAML / OIDC / Token-based login**
* Session timeout ‚Üí 120 minutes

### **3. Admin User**

* Full access
* Only one admin user exists in ULM
* Can act as both admin + retail
* Login: SAML / OIDC / Partner Token
* Session timeout ‚Üí 120 minutes

# ‚≠ê 13. Clean ‚ÄúShort Summary‚Äù (for explaining to someone in 2 mins)

ULM is Vodafone‚Äôs identity, subscription, and user lifecycle platform. It handles registration, authentication, account creation, and subscription assignment for Vodafone Italy and Germany.

Web users authenticate with MFA; retail and admin users authenticate using SAML/OIDC tokens. ULM generates tokens used to access other VF systems.

ULM stores all Web user data in Oracle DB with complex mapping tables like user-subscription, group-subscription, and account mappings. Data is synchronized with CRM/ACRM via APIs and Kafka events. Failed or partial syncs are reprocessed via reconciliation APIs.

Italy uses both legacy and next platform; Germany uses next only with strong token-based architecture. Quarterly migrations move customers from legacy to next using SQL, shell scripts, staging tables, and indexing.

Enriched users allow non-registered users to view offers. ULM cleans or upgrades enriched users when they register. ULM also integrates with CRM, ACRM, ANM, DXL, etc. for real-time validation and provisioning.
